# v0.24.0 — Staff Credit Filtering Design

**Date:** 2026-03-12
**Status:** Approved

---

## Goal

Detect and remove credits-only subtitle lines from ASS, SSA, and SRT files — following the exact same pattern as the existing "Remove HI" tool so no new architectural concepts are introduced.

---

## Architecture

### New module: `backend/credit_remover.py`

Mirrors `hi_remover.py` in structure. No DB access. Pure text transformation.

```
credit_remover.py
  detect_credit_events(subs: SSAFile) -> list[int]
  remove_credits_from_ass(content: str) -> tuple[str, int]   # (cleaned, removed_count)
  remove_credits_from_srt(content: str) -> tuple[str, int]
```

**Detection (4 independent heuristics — any match = credit line):**

1. **Role markers** — regex matching parenthetical role annotations anywhere in the line text:
   `r'\((Translator|QC|Timer|Encoder|Raw Provider|TLC|TS|KFX)\)'`
   Plus standalone thematic labels: `r'^(OP|ED|OP Theme|ED Theme|Staff Roll)$'`

2. **Prefix patterns** — line text (stripped) starts with one of:
   `Cast:`, `Credits:`, `Staff:`, `Translation:`, `Timing:`, `Quality Check:`

3. **Duration heuristic** — event start time > `(total_duration - credit_threshold_sec)`, where
   `total_duration = last dialogue event's end time in the SSAFile`.
   Guard: the line must NOT contain `?` or `!` (prevents stripping genuine dialogue at episode end).

4. **Isolated capitalized names** — line consists of exactly two title-case words and nothing else:
   pattern `^[A-Z][a-z]+ [A-Z][a-z]+$`, max 40 chars. No verb detection — the tight pattern
   itself is the constraint.

Detection is best-effort — false positives are possible. The `dry_run` mode lets users preview before committing.

### New setting: `config.py`

```python
credit_threshold_sec: int = 90  # seconds from end of file to consider credits region
```

Env var: `SUBLARR_CREDIT_THRESHOLD_SEC`. Read via `getattr(settings, "credit_threshold_sec", 90)`.

### New endpoint: `backend/routes/tools.py`

```
POST /api/v1/tools/remove-credits
Body:  { "file_path": "...", "dry_run": false }

200 (applied):
  { "status": "ok", "original_lines": N, "cleaned_lines": N,
    "removed": N, "backed_up": "path.bak" }

200 (dry_run=true):
  { "status": "dry_run", "original_lines": N, "would_remove": N,
    "preview": ["text of event 1", ...] }   # preview capped at 50 items

400:   invalid path, unsupported format (not .ass/.ssa/.srt), or missing file_path
403:   path outside media_path  — checked via is_safe_path(file_path, settings.media_path)
                                   NOTE: arg order is (file_path, base_dir) per security_utils.py
404:   file not found
500:   processing error
```

Response shape mirrors `remove-hi` exactly (`original_lines`, `cleaned_lines`, `removed`).
`backed_up` is additive — `remove-hi` does not return it, but it is safe to add.
Backup created before any write (same `.bak` pattern as `remove-hi`). `dry_run=true` returns
preview of the raw event text strings that would be removed (capped at 50), no file modification.

### Frontend: `SubtitleEditorModal`

One new button **"Remove Credits"** in the Quality Tools toolbar, next to the existing "Remove HI" button. Clicking shows a dry-run preview modal of lines that would be removed, then confirms before calling `removeCredits(filePath)` from `api/client.ts`.

New frontend API function: `removeCredits(filePath: string, dryRun?: boolean)`.
No new hook needed — call directly from the component (same pattern as remove-hi button).

## Testing

**Backend unit tests (`test_credit_remover.py`):**
- Heuristic 1: line with `(Translator)` is detected
- Heuristic 1: standalone `OP Theme` is detected
- Heuristic 2: line starting with `Credits:` is detected
- Heuristic 3: event in credits region (within last 90s) is detected
- Heuristic 3: event in credits region with `?` is NOT removed (guard works)
- Heuristic 3: `credit_threshold_sec` setting is respected (custom value changes boundary)
- Heuristic 4: `John Smith` (two title-case words) is detected
- Heuristic 4: `Hello World!` (punctuation) is NOT detected as credit
- False-positive guard: normal dialogue near end of file is preserved
- `.ssa` files processed same as `.ass` (pysubs2 handles both)
- Backup file created and matches original before modification

**Route tests:**
- `dry_run=true` returns preview, file unchanged
- `dry_run=false` removes credits, creates `.bak`, returns correct counts
- 403 on path outside `media_path`
- 400 on `.vtt` file (unsupported format)

**Frontend tests:**
- `removeCredits()` in `client.ts` sends correct payload
- Remove Credits button renders in SubtitleEditorModal Quality Tools toolbar

---

## Out of Scope

- Machine learning / LLM-based credit detection
- Per-series credit detection configuration (global `credit_threshold_sec` is sufficient)
- ASS style-based detection (credits rarely have dedicated ASS style names)
- Bulk endpoint (single-file only, consistent with remove-hi)
