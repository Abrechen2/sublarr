# v0.31.0 Phase 5 — Frontend Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Apply the same file-size and responsibility principles to the React frontend: split SyncControls.tsx into tab components, split the 2058-line useApi.ts into domain hook files, and add per-section ErrorBoundaries.

**Architecture:** Three independent, additive refactors. SyncControls is split into 4 focused tab components with the orchestrator retaining all state and handlers. useApi.ts becomes a barrel file re-exporting from 6 domain hook files — zero import changes required for existing consumers. ErrorBoundary wraps are added to 3 routes in App.tsx without changing any page component.

**Tech Stack:** React 19, TypeScript, TanStack Query v5, Vitest, ESLint, `tsc --noEmit`

## Task 2: `useApi.ts` Domain Split

**Context:** `frontend/src/hooks/useApi.ts` is 2058 lines with 213 exported hooks. This task splits it into 6 domain files and converts `useApi.ts` into a barrel (re-exports only). **All existing consumers continue to work with zero import changes** because `useApi.ts` will `export *` from all 6 domain files.

**Domain grouping rules — assign each hook to the first matching rule:**

| Domain file | Assign hooks matching these patterns |
|------------|--------------------------------------|
| `useWantedApi.ts` | `useWanted*`, `useScannerStatus`, `useStartBatchProbe`, `useExtractEmbeddedSub`, `useProcessWantedItem`, `useSearchWantedItem`, `useCleanupSidecars` |
| `useTranslationApi.ts` | `useTranslate*`, `useRetranslate*`, `useBackend*`, `useWhisper*`, `useGlossary*`, `useGlobalGlossaryEntries`, `useExportGlossaryTsv`, `useSuggestGlossaryTerms`, `useApproveGlossaryEntry`, `usePromptPreset*`, `useDefaultPromptPreset`, `useTranslationMemory*`, `useDownloadSpecific*`, `useBatchTranslate` |
| `useLibraryApi.ts` | `useLibrary`, `useSeries*`, `useSubtitle*`, `useAdvancedSync`, `useWaveform`, `useExtractAudio`, `useExtractOCR`, `usePreviewOCRFrame`, `useSpell*`, `useVideoChapters`, `useStreamingEnabled`, `useBatchAction` |
| `useProvidersApi.ts` | `useProvider*`, `useScoring*`, `useProviderModifiers`, `useUpdateProviderModifiers`, `useImportScoringPreset`, `useBlacklist`, `useAddToBlacklist`, `useRemoveFromBlacklist`, `useClearBlacklist`, `useLanguageProfile*`, `useCreateProfile`, `useUpdateProfile`, `useDeleteProfile`, `useAssignProfile`, `useHistory*`, `useEpisodeHistory` |
| `useIntegrationApi.ts` | `useMediaServer*`, `useHook*` (hooks system), `useWebhook*`, `useMarketplace*`, `useInstalledPlugins`, `useInstall*Plugin`, `useUninstallBrowsePlugin`, `useRefreshMarketplace*`, `useAnidb*`, `useCompatCheck`, `useExtendedHealth*`, `useExportIntegrationConfig*`, `useBazarr*` |
| `useSystemApi.ts` | Everything else: health, stats, config, jobs, batch status, tasks, logs, statistics, quality trends, health check/fix, backup, log rotation, standalone, watched folders, cleanup, duplicates, orphaned, api keys, notifications, quiet hours, config export/import, global search, filter presets, search (interactive, episode, all wanted), `useSupportedLanguages` |

**Process for the split:**
1. Read the full `useApi.ts`
2. For each domain file: find all matching hooks, collect their implementations, collect the `@tanstack/react-query` imports they use (useQuery/useMutation/useQueryClient/keepPreviousData), collect the `@/api/client` functions they import
3. Write each domain file with its own imports + hook implementations
4. Replace `useApi.ts` content with barrel exports
5. Run `tsc --noEmit` — fix any errors (missing imports, wrong groupings)
