# v0.25.3 — List Virtualization Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace client-side pagination in Library (table view) and Wanted with `@tanstack/react-virtual` virtual scrolling for smooth, performant rendering of large lists.

**Architecture:** Use the **padding-row technique** — wrap the existing `<table>` in a fixed-height overflow container, inject spacer `<tr>` elements at top/bottom of `<tbody>`, and render only the rows visible in the viewport. This preserves all `<table>/<tr>/<td>` semantics and requires no div-based layout conversion. `@tanstack/react-virtual` is already installed.

**Tech Stack:** `@tanstack/react-virtual` v3 (`useVirtualizer`), React 19, TypeScript — no new dependencies.

### Task 2: Extract `VirtualLibraryTable` component

**Files:**
- Create: `frontend/src/components/library/VirtualLibraryTable.tsx`
- Modify: `frontend/src/pages/Library.tsx` (remove `LibraryTable`, wire `VirtualLibraryTable`)

**Context:**
`Library.tsx` currently renders a `LibraryTable` component that outputs a `<table>` with `<tbody>` → `<tr>` rows via `LibraryTableRow`. Pagination happens at the page level with a `Pagination` component and `PAGE_SIZE = 25`. The plan: remove pagination for table view, add virtual scroll using the padding-row technique.

The `LibraryTable` component already lives in `Library.tsx` (not a separate file) and accepts:
- `items: (SeriesInfo | MovieInfo)[]` — the full filtered+sorted array (currently paginated slice)
- `type: Tab` (`'series'` | `'movies'`)
- `profiles`, `onRowClick`, `onProfileChange`, `sortKey`, `sortDir`, `onSort`, `t`, `selectedSeries`, `onToggleSeries`, `onSelectAll`, `onClearSelection`

Column count:
- Series tab: 5 columns (checkbox, title, missing, profile, episodes)
- Movies tab: 3 columns (title, profile, status)

**The padding-row pattern:**

```tsx
import { useRef } from 'react'
import { useVirtualizer } from '@tanstack/react-virtual'

// Inside VirtualLibraryTable:
const parentRef = useRef<HTMLDivElement>(null)
const rowVirtualizer = useVirtualizer({
  count: items.length,
  getScrollElement: () => parentRef.current,
  estimateSize: () => 40,   // px — approximate row height
  overscan: 8,              // extra rows rendered outside viewport
})
const virtualItems = rowVirtualizer.getVirtualItems()
const paddingTop = virtualItems.length > 0 ? virtualItems[0].start : 0
const paddingBottom =
  virtualItems.length > 0
    ? rowVirtualizer.getTotalSize() - virtualItems[virtualItems.length - 1].end
    : 0
const colSpan = isSeries ? 5 : 3
```

In JSX:
```tsx
<div ref={parentRef} style={{ height: 'calc(100vh - 280px)', overflowY: 'auto' }}>
  <table className="w-full">
    <thead>...</thead>
    <tbody>
      {paddingTop > 0 && (
        <tr aria-hidden="true">
          <td colSpan={colSpan} style={{ height: paddingTop, padding: 0 }} />
        </tr>
      )}
      {virtualItems.map(({ index }) => (
        <LibraryTableRow key={items[index].id} item={items[index]} ... />
      ))}
      {paddingBottom > 0 && (
        <tr aria-hidden="true">
          <td colSpan={colSpan} style={{ height: paddingBottom, padding: 0 }} />
        </tr>
      )}
    </tbody>
  </table>
</div>
```

### Task 3: Virtualize Wanted list

**Files:**

**Context:**
The Wanted page uses server-side pagination via `useWantedItems(page, 50, ...)`. For virtualization we need all items loaded at once. Strategy: use `perPage = 9999` to fetch all matching items in one request; the backend `/wanted` endpoint supports this — it just queries more rows. The virtualizer then renders only visible rows.

Wanted table structure is more complex than Library (many columns, expandable rows, action buttons). The virtualization is applied only to the main list rows, not to expanded detail panels.

**Key concern:** Wanted rows have variable height when the details panel is expanded (`ChevronUp`/expanded state). The `estimateSize` should be the collapsed row height (~42px). When rows expand, the virtualizer's measurement will be off. To handle this:
- Use `rowVirtualizer.measureElement` ref on each row for dynamic height measurement.
- `useVirtualizer({ count, getScrollElement, estimateSize: () => 42, measureElement })`

**Simpler alternative:** Since only one row is typically expanded at a time, estimate height at 42px and accept minor scroll-bar jitter on expand. This is the pragmatic approach for v0.25.3.

  ```tsx
  import { useRef } from 'react'
  import { useVirtualizer } from '@tanstack/react-virtual'

  interface VirtualWantedTableProps {
    count: number
    renderRow: (index: number) => React.ReactNode
    colSpan: number
    estimatedRowHeight?: number
  }

  /**
   * Generic virtual scroll wrapper for table-based lists.
   * Uses the padding-row technique: injects spacer <tr> at top/bottom of <tbody>,
   * renders only visible rows. Keeps <table> DOM structure intact.
   */
  export function VirtualWantedTable({
    count,
    renderRow,
    colSpan,
    estimatedRowHeight = 42,
  }: VirtualWantedTableProps) {
    const parentRef = useRef<HTMLDivElement>(null)
    const rowVirtualizer = useVirtualizer({
      count,
      getScrollElement: () => parentRef.current,
      estimateSize: () => estimatedRowHeight,
      overscan: 8,
    })
    const virtualItems = rowVirtualizer.getVirtualItems()
    const paddingTop = virtualItems.length > 0 ? virtualItems[0].start : 0
    const paddingBottom =
      virtualItems.length > 0
        ? rowVirtualizer.getTotalSize() - virtualItems[virtualItems.length - 1].end
        : 0

    return (
      <div ref={parentRef} style={{ height: 'calc(100vh - 300px)', overflowY: 'auto' }}>
        {/* Table header is rendered by the caller OUTSIDE this component */}
        {paddingTop > 0 && (
          <tr aria-hidden="true">
            <td colSpan={colSpan} style={{ height: paddingTop, padding: 0 }} />
          </tr>
        )}
        {virtualItems.map(({ index }) => renderRow(index))}
        {paddingBottom > 0 && (
          <tr aria-hidden="true">
            <td colSpan={colSpan} style={{ height: paddingBottom, padding: 0 }} />
          </tr>
        )}
      </div>
    )
  }
  ```

  ```tsx
    const virtualItems = rowVirtualizer.getVirtualItems()
    const paddingTop = virtualItems.length > 0 ? virtualItems[0].start : 0
    const paddingBottom =
      virtualItems.length > 0
        ? rowVirtualizer.getTotalSize() - virtualItems[virtualItems.length - 1].end
        : 0
    return { parentRef, virtualItems, paddingTop, paddingBottom }
  }
  ```

## Notes for Executor

**Sticky header:** Add `position: 'sticky', top: 0, zIndex: 1` to the `<thead>` element so column headers stay visible while scrolling. Background must be set explicitly (e.g. `backgroundColor: 'var(--bg-elevated)'`), otherwise rows will scroll under a transparent header.

**Scroll reset on filter/sort:** After filter or sort changes in Library, call `parentRef.current?.scrollTo({ top: 0 })` inside a `useEffect` that watches `processedItems`. Pass `parentRef` up from `VirtualLibraryTable` via a ref callback or expose a `scrollToTop` handle via `useImperativeHandle` if needed. Simpler: just add `key={sortKey + sortDir + searchQuery + statusFilter + profileFilter}` to `VirtualLibraryTable` to remount on filter change — this resets scroll automatically.

**Wanted expandable rows:** Row height estimation is 42px (collapsed). When a row expands to show detail, the virtualizer will underestimate total height causing minor scroll jitter. This is acceptable for v0.25.3. If it proves annoying, use `rowVirtualizer.measureElement` on each row ref in a future patch.

**Testing on Windows + Node 24:** Vitest doesn't run locally (see MEMORY). TypeScript + lint are the local checks. CI (Ubuntu + Node 20) runs the test suite.
