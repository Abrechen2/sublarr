# v0.31.0 Phase 4 — Architecture Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Three structural improvements that reduce coupling: provider metadata centralized in a registry, settings exposed via grouped views, and singletons registered in `app.extensions` for lifecycle visibility.

**Architecture:** Purely additive or internal refactors — no API changes, no new endpoints, no behavior changes. Task 1 extracts three in-class dicts into a new file. Task 2 adds read-only view classes that delegate to existing flat Settings fields. Task 3 makes singleton creation visible via `app.extensions` while keeping module-level globals as a fallback for non-request contexts.

**Tech Stack:** Python 3.12, Flask, Pydantic `BaseSettings`, pytest

  rate_limit  : (max_requests, window_seconds) — 0/0 means no limit

    def _get_rate_limit(self, provider_name: str) -> tuple[int, int]:
        """Get rate limit for a provider: (max_requests, window_seconds).

        Prefers class attribute, falls back to registry, then (0, 0).
        """
        cls = _PROVIDER_CLASSES.get(provider_name)
        if cls:

```python
    def _get_timeout(self, provider_name: str, all_stats: dict | None = None) -> int:
        """Get timeout for a provider (seconds).

        Priority:
        1. Dynamic timeout computed from historical avg_response_time_ms
        2. Class attribute (provider-specific hardcoded timeout)
        3. Registry (PROVIDER_METADATA)
        4. Global provider_search_timeout setting
        """
        # 1. Dynamic timeout from stats
        if all_stats and provider_name in all_stats:
            dynamic = self._compute_dynamic_timeout(provider_name, all_stats[provider_name])
            if dynamic:
                return dynamic
        # 2. Class attribute
        cls = _PROVIDER_CLASSES.get(provider_name)
        if cls:
            class_timeout = getattr(cls, "timeout", 0)
            if class_timeout > 0:
                return class_timeout
        # 3. Registry, 4. Global setting
        meta = PROVIDER_METADATA.get(provider_name, {})
        return meta.get("timeout", self.settings.provider_search_timeout)
```

class _SettingsView:
    """Base for read-only Settings group views.

    Delegates attribute access to the parent Settings instance for fields
    declared in the subclass ``_fields`` tuple. Raises ``AttributeError``
    for any field not in ``_fields``.
    """

    _fields: tuple[str, ...] = ()

    def __init__(self, settings: Settings) -> None:
        object.__setattr__(self, "_s", settings)

    def __getattr__(self, name: str):
        if name in self._fields:
            return getattr(self._s, name)
        raise AttributeError(f"{type(self).__name__!r} has no attribute {name!r}")

## Task 3: Singleton Lifecycle via `app.extensions`

**Context:** `WantedScanner` and `ProviderManager` are application-lifetime singletons currently stored only in module-level globals (`wanted_scanner._scanner`, `providers._manager`). This makes it hard to mock them in tests — callers must patch at the import level. Registering them in `app.extensions` lets tests inject mocks by writing to `current_app.extensions["wanted_scanner"]`, a simpler and less fragile mechanism.

**Change summary:**
1. `wanted_scanner.get_scanner()` / `invalidate_scanner()`: check and update `app.extensions` when an app context is active.
2. `providers.get_provider_manager()` / `invalidate_manager()`: same pattern with three private helpers (`_has_app_context`, `_get_from_extensions`, `_set_in_extensions`, `_pop_from_extensions`).
3. `app.py`: unconditionally populate `app.extensions` with both singletons right before `return app`.

**Design note:** Scheduler threads run without an app context — `has_app_context()` returns `False` there, so the module-level global path is used. No change in scheduler behaviour. After `invalidate_*()`, the next call to `get_scanner()` / `get_provider_manager()` inside an app context creates a new singleton and re-populates `app.extensions` (self-healing).

**Files:**

- Modify: `backend/providers/__init__.py` (lines 89–107, add 4 private helpers)
- Modify: `backend/app.py` (add 4 lines before `return app`)

def get_scanner():
    """Get or create the singleton WantedScanner (thread-safe).

    When called inside a Flask app context, the result is stored in and
    retrieved from ``app.extensions["wanted_scanner"]`` — this lets tests
    inject a mock by writing to that key. Falls back to a module-level
    global when no app context is available (e.g. scheduler threads).
    """
    global _scanner
    # Use app.extensions when an app context is active
    if _has_flask_app_context():
        scanner = _get_flask_extension("wanted_scanner")
        if scanner is not None:
            return scanner
    if _scanner is None:
        with _scanner_lock:
            if _scanner is None:
                _scanner = WantedScanner()
    # Re-populate extensions if inside an app context (self-healing after invalidation)
    if _has_flask_app_context():
        _set_flask_extension("wanted_scanner", _scanner)
    return _scanner


def invalidate_scanner():
    """Reset the scanner singleton (call after config changes)."""
    global _scanner
    if _scanner:
        _scanner.stop_scheduler()
    _scanner = None
    _pop_flask_extension("wanted_scanner")


def _has_flask_app_context() -> bool:
    try:
        from flask import has_app_context

        return has_app_context()
    except ImportError:
        return False


def _get_flask_extension(key: str):
    try:
        from flask import current_app

        return current_app.extensions.get(key)
    except RuntimeError:
        return None


def _set_flask_extension(key: str, value) -> None:

        pass


def _pop_flask_extension(key: str) -> None:
